ProSlides

En interaktiv presentasjonsplattform med sanntidsfunksjonalitet – polls, ordskyer, tegning og live-samarbeid. Bygget som bachelorprosjekt ved USN, våren 2026.

Se kildekoden

Om prosjektet

ProSlides er en fullstack webapplikasjon som lar forelesere og presentatører lage interaktive presentasjoner med sanntidsfunksjonalitet. Publikum kan delta i polls, ordskyer og andre interaktive elementer direkte fra sine enheter.

Monorepo-struktur

Prosjektet er organisert som en monorepo med to hovedmapper:

Bachelor_Gruppe1/
├── backend/  Rails 8.1 API
├── frontend/ React + Vite SPA
├── scripts/
└── README.md

Nøkkelfunksjoner

  • Lag og rediger presentasjoner med en visuell editor
  • Sanntids polls og ordskyer via WebSockets
  • OAuth-innlogging (Google, GitHub)
  • Tegne-canvas med Fabric.js
  • Drag-and-drop slide-organisering
  • Deploy med Kamal til produksjon

Tidslinje

Prosjektet er delt opp i fire sprinter, tilpasset tidsrammen.

S1 – Start · Uke 5–7

Foranalyse

Sette i gang med prosjektet. Oppsett av IDE, rammeverk og utviklingsmiljø. Sette seg inn i Ruby og Rails. PostgreSQL-database satt opp med riktig tilkobling.

  • Oppsett av Rails 8 API + React/Vite monorepo
  • PostgreSQL-database med migrasjoner
  • Git-flyt og prosjektstruktur definert
  • Læringsperiode for Ruby og Rails
S2 – Design · Uke 8–10

UI, UX og datastrukturer

Etablere fast UI-struktur. Sikre interaktiv og intuitiv UX med universell utforming. Definere datastrukturer og modeller.

  • Komponentbibliotek med shadcn/ui + Tailwind
  • Databasemodeller: User, Presentation, Slide, Poll
  • Ruting og navigasjonsstruktur
  • Universell utforming (WCAG)
S3 – Implementasjon · Uke 11–17

Editor, interaktivitet og sanntid

Editoren er kjernen i prosjektet og får en hel sprint. All funksjonalitet på plass: verktøylinje, polls, ordsky, slides, tegning. Strukturert og oversiktlig side med tilhørende funksjoner.

  • Visuell slide-editor med Fabric.js-canvas
  • Drag-and-drop slide-organisering (dnd-kit)
  • Sanntids polls og ordskyer via Action Cable
  • OAuth-innlogging (Google, GitHub)
  • JWT-autentisering og sessions
  • Live presentasjonsmodus med deltaker-kobling
S4 – Testing · Uke 18–20

Feilretting og stabilisering

Brukertesting av nettsiden, brukerhistorier og selvtesting. Feil og mangler fikses fortløpende. Stabilisering av produktet.

  • E2E-testing med Playwright
  • Brukertesting med reelle testpersoner
  • Testing basert på brukerhistorier
  • Feilretting og ytelsesforbedring
  • Deploy-testing med Docker + Kamal
Rapport · Uke 5–21

Rapportskriving

Gjøres gjennom hele prosjektets levetid, parallelt med utvikling. Dokumenterer prosess, metode, resultater og refleksjon.

Levering · Uke 21 (19. mai)

Endelig produkt

Siste optimalisering, feilretting og endringer. Levering av ferdig produkt, rapport og prosjektoversikt.
